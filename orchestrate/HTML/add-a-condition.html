You can add one or more conditional checks that a user must satisfy to move to the next step. You can filter users based on the past events they performed, their user attributes or the attributes of the entry event, which qualified the user for the workflow.

   
The Performed an event criteria lets you filter users based on the events
they performed in the recent past - that is, up to 48 hours. 

  1. Click + and choose Add a Condition.

  2. Add the condition by choosing the event name and setting the number of occurrences within a specified time limit.
  3. Click Add Condition. The condition is added to the workflow builder.

Look Back Condition and When to Add One

Zeotap helps you understand your user’s behaviour in the recent past and then take the necessary follow-up actions that best suit your business. To do this, the first step is to define which events and attributes you're interested in filtering. The second step is to define how recently and how frequently a user should have done that action. This lets you power more personalised recommendations, push or email notifications.

For example, you want to identify users who are abandoning their carts. In such a scenario, you first track when someone adds a product to their cart. Then, after waiting for some time you look back and see if they made the purchase or not.

Entry Event Attributes Filter

When you perform a specific event that meets the 'Entry Event Condition', you become eligible for a particular workflow. You can use the 'Have Entry Event Attributes' filter to refer to that event again and use its attributes for filtering. Ensure that you use the same event name that have you used in the entry condition, as this filter always refers to the attributes of the entry event that triggered the workflow.
